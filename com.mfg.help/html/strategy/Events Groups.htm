Events Groups

The event groups allow us to combine several events together and form more complex rules to be executed. The combined events can be Atomic Events or can also be one of the defined events groups.

SORTED Events Group

This event is to be triggered when all the events it contains are triggered in the order they come. The most common usage is to form a pattern when we first execute the entry and then the exit.

UNSORTED Events Group

The UNSORTED event is triggered when all its events are triggered no matter in which order they get triggered.

OR Events Group

The OR event is triggered when one of its events is triggered.

XOR Events Group

The XOR event generally will combine events groups and when one of then starts to match its parts this event will focus on that particular sub event and won’t check anymore the rest.

AND Events Group

The AND event is triggered when all of its events are triggered at the same time.

OAND Events Group

We can say that the OAND event does a check that is very similar to the check we do in the OR event, but when the event gets triggered, the non-triggered events are added as Threads so they can be checked on later ticks till the exit comes and they are removed.
