Atomic Events

Atomic events are the simplest form of events that can be expected in our trading context, they can be used in their simple form, or they can be combined in order to obtain more complex patterns to be checked.

The have a common basic life cycle that allows them to be activated through time. Let’s say they start by a preinit process where they collect some general information and that happens when the pattern is created. Following the preinit process we have the init process where information regarding events parameters if needed is taken to compute the conditions that trigger the event. After that we keep on checking if the event is triggered till the conditions are the right ones and we can say the event is triggered. If the event has a “limit to Sw0” flag on, that means the triggering of the event needs to occur before the next threshold of that scale comes otherwise it will never be triggered.

THRESHOLD Event

The threshold event (TH) is one of the simplest events we can define and it occurs when the Indicator detects at a certain scale that the swing channel is changing the direction. The event is triggered at that moment or it can also be set to be triggered after a specific number of skipped thresholds, so if we set 2 in the threshold to skip parameter the event will be triggered after it skips 2 thresholds, that is on the 3rd one we get during the check process.

SC and RC Events

The SC(RC) Event is takes place when the price touches the SC(RC) line defined by the indicator as the Bottom Regression Line when channel is up and the Top Regression Line when channel direction is down (we invert directions for RC). The event is triggered every time the price touches the line.

On Figure 2‑2, prices near the SC label that touches the top yellow line (Top Regression Line because after TH2 the direction of the swing is down) are the SCs. After TH2 we have an RC because price touches the Bottom Regression Line in a down swing and also after TH3 because price touches the Top Regression Line in an up-swing.

CL Xover

The Central Line Crossover (CL Xover) is the event triggered when the price crosses the Central Regression Line. The event can occur contrarian to the direction of the swing, or non-contrarian to it. For example if we are in a down swing, and price is below the Central Line, at the moment it goes above the central Line, we can say that a CL Xover contrarian to the direction of the swing.   In the Figure 2  we can see that ticks in red circles are some of the points where a contrarian CL Xover is activated, because they are the first tick after the cross down (notice we are in an up-swing) of the magenta central regression line. Yellow ones are non-contrarian CL Xover points

Sw Ratio

TH Ratio event represents the event that is triggered when at the first time we realize that the ratios between swings fall in a certain range defined by the user by setting the values for lower and upper bounds. Dimension parameter will determine the swings that will be considered in the ratios, if dimension is 1, we only consider ratio s(0)/s(-1) otherwise we consider all possibilities of ratios given by a swing and the swings before that one. Max dimension is 5. Usually the event is triggered at a new TH and keeps on active during that swing.

For example if we want to get activated when s(-1) is lower than s(-2) we would want s(-1)/s(-2) in range 0..1, in the Figure 3  is shown where the activations of the event takes place.

C/NC Event

Given a base scale (later in 3.1 we analyze how that base scale is computed) the C/NC Event represents the event when the base scale is contrarian to the specified scales and non-contrarian to the specified scales. The user can specify the Contrarian and Non-Contrarian Scales by adding them to the corresponding list in the editor. It is usually triggered when a TH of one of the scales comes and we get the right C/NC pattern and it keeps on active during the swing.

Profit/Loss Event

The event allows us to consider another event like a TH only if the specified trades that are still open are in profit or loss.
