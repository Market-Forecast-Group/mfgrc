Commands

Possible commands in this strategy are the entries, and exits. When we initialize them the corresponding orders are sent to the market and the event status is triggered.

Entry Event

When the entry event is initialized sends to the market an order (market or limit family), and then when the command is executed we change the price of the previously sent order so it can be executed as soon as possible. The entry order can also have a Stop Loss Simple Protection at a certain scale and/or a limit child to be able to close the position later with a Profit Take or a Stop Loss.

Exit Event

When we execute an exit command we usually modify the price of a previously sent child to close a position, but in case that child is not present we will need to send a parent order routed to the opposite account. Exits can be from the market family (Stop Loss) or from Limit Family (Profit Take). The exit command can also be associated to several entries to be able to close all the specified opened positions.

Conditional Commands

Conditional commands are an event wrap to deal with commands, when a certain event (precondition) is triggered the corresponding command is executed. When we start checking the precondition the corresponding command is initialized.
